# F# pros and cons

F# pros:

-  Easy to learn
-  Easy to write
-  Easy to debug
-  Easy to test
-  Easy to build
-  Easy to share
-  Easy to document
-  Easy to learn from
-  Easy to use
-  Easy to build on

F# cons:
- No type system
- No static typing
- No compile time type checking
- No compile time type inference
- No compile time type checking
- No compile time type checking
- No compile time type checking
